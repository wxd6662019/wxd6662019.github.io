useful-git
2024/9/2

有用的命令

1. 基本操作

查看Git当前全局配置

git config --global --list
#如果没有就先添加
git config --global user.name "xxx"
git config --global user.email "xxx"

  • 初始化仓库

    git init

  • 克隆仓库：把远程仓库拷贝到你指定的文件名下

    git clone <repository-url> 文件名

  • 清楚git缓存

    git credential-cache exit

2. 查看状态

  • 查看当前状态（万能命令）

    #告知你下一步该做的命令
git status

  • 查看提交历史

    git log --oneline --graph --all

3. 文件操作

  • 添加文件到暂存区

    git add <file>
git add .

  • 从暂存区移除文件

    #从暂存区移除文件
git reset <file>
# 丢弃没有暂存的更改，将文件恢复至最近一次提交
git restore <file>
#将文件从工作区和暂存区中删除
git rm <file>

  • 查看工作区与暂存区之间的文件差异

    git diff

4. 提交更改

  • 提交更改

    git commit -m "Commit message"

5. 分支管理

  • 查看分支

    git branch

  • 创建新分支

    git branch <branch-name>

  • 切换分支

    git checkout <branch-name>
#如果不想要这个分支，切回master，删除掉新分支
git branch master
git branch -D branch-name

  • 合并分支

    #如果想要这个分支，则合并，但是有几种情况需要考虑
#快进，在dev分支上修改，切回master，让它去合并dev，删除dev
git merge <branch-name>
#分叉，两个分支上有不同的修改，可以轻松合并，head指向公共下游
#冲突，两个分支上有相同的修改，无法自动合并，需要解决冲突后合并
git merge --abort 
#重要！任何时候发现冲突，调用git status，查看提示

    ps：工作中，选择一个分支，如master，在历史记录中只进不退

6. 远程操作

  • 查看远程仓库（列出和你关联的远程仓库的信息）

    git remote -v

  • 添加远程仓库

    git remote add origin <repository-url>

  • 推送到远程仓库

    #要求：本地分支是origin/branch-name 的正下游
git push origin <branch-name>

#以master为例，如果本地master分支和远程master分支在不同路线
git fetch origin <branch-name> #拉取远程master分支
#将本地master分支通过merge快进到远程master分支
git merge origin/master #此时应该处在本地分支上
#此时本地master分支和远程master分支处在相同路线上

#注意：fetch拿到的origin/master只能被merge不能被切换

    注意：只有同名分支才可以做 push/fetch 操作，且本地分支相对于远程分支是正下游分支才可推送成功

  • 拉取远程更新

    #此操作相当于前面的整合，即先fetch，再merge
git pull origin <branch-name>

  • 重新设置远程仓库

    git remote set-url origin https://github.com/你的用户名/新的仓库名.git

7. 其他实用命令

  • 撤销上一次提交（保留更改）

    git reset --soft HEAD~1

  • 强制推送

    git push --force

  • 查看已追踪的文件

    git ls-files

8. 标签管理

  • 创建标签

    git tag <tag-name>

  • 推送标签到远程

    git push origin <tag-name>

9. 当你在进行合并操作时，若当前工作区中有未提交的更改，这些更改会被合并操作覆盖。Git 为了防止数据丢失，会阻止这个操作

解决方法

  1. 提交更改：

    • 如果你希望保留当前的更改，可以先将它们提交：
      git add .
git commit -m "Your commit message"

  2. 暂存更改：

    • 如果你不想立即提交这些更改，可以使用 git stash 命令将它们暂存：
      git stash
    • 暂存后，你可以执行合并操作。完成后，可以通过以下命令恢复暂存的更改：
      git stash pop

  3. 丢弃更改：

    • 如果你确定不需要当前的更改，可以使用以下命令丢弃它们：
      git checkout -- fileName
    • 注意，这将永久删除未提交的更改，请谨慎操作。
