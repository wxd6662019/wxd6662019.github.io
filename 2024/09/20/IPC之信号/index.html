IPC之信号

2024/9/20 理论 Linux

1. 信号的定义

  • 信号是一种异步通知机制，用于向进程发送事件通知。这些事件可以是外部事件（如用户按键、定时器到期）或内核事件（如进程错误）。

Each signal has a current disposition, which determines how the process behaves when it is delivered the signal.

2. 信号的类型

  • 常见的信号包括：
    • SIGINT：中断信号，通常由 Ctrl+C 发送。
    • SIGTERM：终止信号，要求进程终止。
    • SIGKILL：强制终止信号，不能被捕获或忽略。
    • SIGUSR1 和 SIGUSR2：用户定义的信号，可以用于自定义事件处理。

3. 信号的发送

  • 信号可以由以下方式发送：
    • 命令行：使用 kill 命令发送信号。kill -l 查看所有信号
    • 系统调用：使用 kill() 或 raise() 函数在程序内部发送信号。

4. 信号的处理

  • 进程可以选择如何响应信号：
    • 默认处理：每种信号都有默认的处理方式（如终止进程）。
    • 自定义处理：使用 signal() 或 sigaction() 函数可以注册自定义的信号处理函数。

前置知识

  1. 屏蔽集合 (Blocked Signal Set) 定义：屏蔽集合用于指定在某个进程中哪些信号是被阻塞的。被阻塞的信号在进程执行期间不会被处理。 用途： 保护临界区：在进行关键操作时，可以屏蔽某些信号，以防止信号打断这些操作。 信号的临时阻止：使用 sigprocmask() 可以动态地修改进程的屏蔽集合。
  2. 未决集合 (Pending Signal Set) 定义：未决集合包含了已经发送但尚未被进程处理的信号。当进程处于阻塞状态时，信号仍然可以被发送到该进程，这些信号会进入未决集合。 用途： 记录信号状态：未决集合确保即使信号被阻塞，进程仍然可以在稍后处理这些信号。 通过 sigpending() 可以查询当前未决集合中的信号。
  3. 信号处理流程 当信号发送到一个进程时： 如果该信号在屏蔽集合中，信号不会被处理，进入未决集合。 如果该信号不在屏蔽集合中，进程会根据信号的处理方式（默认或自定义处理程序）立即处理该信号。

5. 信号的阻塞

  • 进程可以选择阻塞某些信号，使用 sigprocmask() 函数进行信号屏蔽。这可以防止在处理某些关键操作时接收到信号。

6. 信号的排队

  • 某些信号（如 SIGUSR1 和 SIGUSR2）可以被排队，如果同一个信号在短时间内多次发送到同一进程，后续信号会被丢弃，除非使用实时信号（SIGRTMIN 到 SIGRTMAX）来确保排队处理。

7. 信号的优势和局限

  • 优势：
    • 简单且快速的通信机制。
    • 适合用于异步事件处理。
  • 局限：
    • 信号的处理是异步的，可能导致竞态条件。
    • 不能传递复杂数据，只能传递信号标识。
    • 在信号处理过程中，可能会影响程序的正常执行。

总结

信号是 Unix/Linux 系统中一种重要的进程间通信机制，用于异步通知和事件处理。理解信号的使用和管理对于编写健壮的多进程程序至关重要。
