2024/9/6 命令 Linux

lsof（List Open Files）是一个用于列出当前系统中打开文件的命令。由于在 Linux 和 Unix 系统中，几乎所有内容（包括设备、网络连接和进程）都被视为文件，因此 lsof 是一个非常强大的工具。

lsof 的基本用法

  1. 查看所有打开的文件

    lsof

    输出当前系统中所有打开的文件及其相关信息。

  2. 查看特定进程的打开文件

    lsof -p <PID>
    • <PID>：指定进程 ID。
    • 示例：lsof -p 1234 将列出进程 ID 为 1234 的所有打开文件。

  3. 查看特定用户的打开文件

    lsof -u <username>
    • <username>：指定用户的用户名。
    • 示例：lsof -u kola 将列出用户 kola 打开的所有文件。

  4. 查看特定文件的使用情况

    lsof <filename>
    • 示例：lsof /etc/passwd 将显示所有正在使用 /etc/passwd 文件的进程。

  5. 查看网络连接

    lsof -i
    • 显示所有打开的网络连接和监听端口。

  6. 过滤特定协议的网络连接

    lsof -i TCP
    • 只列出 TCP 连接。

  7. 查看特定端口的使用情况

    lsof -i :<port>
    • <port>：指定端口号。
    • 示例：lsof -i :80 将显示使用 80 端口的所有进程。

  8. 查看被删除但仍在使用的文件

    lsof | grep '(deleted)'
    • 可以帮助找出已经被删除但仍被进程打开的文件。

  9. 显示进程的详细信息

    lsof -V
    • 显示版本信息和帮助。

实用示例

  • 找出哪个进程占用了特定端口：

    lsof -i :8080

  • 查看某个用户的网络活动：

    lsof -u kola -i

  • 查找所有打开的文件并重定向到文件：

    lsof > open_files.txt

主组和附加组：

假设有一个用户 alice，其主组为 users，并且她还属于附加组 dev 和 admin。这意味着：

  • 当 alice 创建文件时，默认的组是 users。
  • alice 可以访问属于 dev 和 admin 组的资源。

root@hostname : @符后面跟着主机的名字，root 则是当前登录的用户

hostnamectl set-hostname 新的主机名 可以更改当前服务器的主机名

ps显示当前运行的进程，a显示所有用户的进程，u以用户格式显示进程信息，x显示没有控制终端的进程

运行 ps aux | grep ssh-agent 后，输出通常包含以下列：

  • USER：进程的所有者。
  • PID：进程ID。
  • %CPU：进程占用的CPU百分比。
  • %MEM：进程占用的内存百分比。
  • VSZ：进程使用的虚拟内存大小（以KB为单位）。
  • RSS：进程使用的物理内存大小（以KB为单位）。
  • TTY：进程关联的终端。
  • STAT：进程状态（如运行、睡眠等）。
  • START：进程启动时间。
  • TIME：进程使用的CPU时间。
  • COMMAND：启动该进程的命令及其参数。

by the way , 在Windows上则是

ps | Where-Object { $_.ProcessName -like "*ssh-agent*" }

使用别名可以缩短较长的命令

打开文件：nano ~/.bashrc

添加别名：alias gs='git status'

保存退出并运行：source ~/.bashrc

在Unix/Linux环境的命令行界面下的快捷键

Ctrl+A ：表示将光标移动到最左端
Ctrl+E：光标移动到最右端
Ctrl+U：删除光标前所有输入
Ctrl+K：删除光标后所有输入
Ctrl+L：清屏（power shell中也可用）
Ctrl+C：中止当前进程
Ctrl+D：在输入时发送EOF

解压操作：tar -zxvf filename.tar.gz

-z: 使用gzip格式 ，x表示解压，v表示看，f表示指定文件名
