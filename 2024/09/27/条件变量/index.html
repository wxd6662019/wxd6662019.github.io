条件变量题目

2024/9/27 理论 Linux

基本概念

问: 什么是条件变量？它通常用于解决什么样的问题？

答：条件变量是一种用于线程间同步的机制，允许一个线程等待某个条件发生，而其他线程可以通知这个条件已经发生，通常见于生产者消费者问题，读者写者问题

问: 在使用条件变量时，为什么需要同时使用互斥锁（mutex）？

答：条件变量的等待和通知操作必须在互斥锁保护下进行，以确保在检查和修改共享状态时的线程安全，这样可以避免在检查条件和等待之间的竞争条件

问: 条件变量的 pthread_cond_signal 和 pthread_cond_broadcast 有什么不同？在什么情况下应该使用它们？

答：只希望唤醒一个线程时使用前者，而当多个线程需要处理同一条件时使用后者

问: 使用条件变量时，常见的错误有哪些？如何避免？

答:

  1. 没有在互斥锁保护下调用 pthread_cond_wait 和 pthread_cond_signal：这会导致数据竞争和未定义行为。确保在锁保护下进行这些操作。
  2. 在条件变量上没有正确检查条件：可能会导致虚假唤醒。使用 while 循环来检查条件，而不是 if 语句。
  3. 忘记初始化条件变量和互斥锁：在使用之前，必须初始化它们。

问: 编写一个简单的生产者-消费者示例，使用条件变量来同步生产者和消费者线程

#include <stdio.h>
#include <stdlib.h>
#include <pthread.h>
#include <unistd.h>

#define BUFFER_SIZE 5

int buffer[BUFFER_SIZE];
int count = 0;

pthread_mutex_t mutex;
pthread_cond_t cond_empty;
pthread_cond_t cond_full;

void* producer(void* arg) {
    for (int i = 0; i < 10; i++) {
        pthread_mutex_lock(&mutex);
        
        while (count == BUFFER_SIZE) { // 如果缓冲区满，等待
            pthread_cond_wait(&cond_empty, &mutex);
        }

        // 生产者生产数据
        buffer[count++] = i;
        printf("Produced: %d\n", i);
        
        pthread_cond_signal(&cond_full); // 通知消费者
        pthread_mutex_unlock(&mutex);
        sleep(1);
    }
    return NULL;
}

void* consumer(void* arg) {
    for (int i = 0; i < 10; i++) {
        pthread_mutex_lock(&mutex);
        
        while (count == 0) { // 如果缓冲区空，等待
            pthread_cond_wait(&cond_full, &mutex);
        }

        // 消费者消费数据
        int item = buffer[--count];
        printf("Consumed: %d\n", item);
        
        pthread_cond_signal(&cond_empty); // 通知生产者
        pthread_mutex_unlock(&mutex);
        sleep(1);
    }
    return NULL;
}

int main() {
    pthread_t prod, cons;
    pthread_mutex_init(&mutex, NULL);
    pthread_cond_init(&cond_empty, NULL);
    pthread_cond_init(&cond_full, NULL);

    pthread_create(&prod, NULL, producer, NULL);
    pthread_create(&cons, NULL, consumer, NULL);

    pthread_join(prod, NULL);
    pthread_join(cons, NULL);

    pthread_mutex_destroy(&mutex);
    pthread_cond_destroy(&cond_empty);
    pthread_cond_destroy(&cond_full);

    return 0;
}
