线程
理论 2024/9/20

在 Unix/Linux 系统中，线程相关的系统调用主要用于创建、管理和控制线程。以下是一些常见的线程相关系统调用及其功能：

1. pthread_create()

  • 功能：创建一个新线程。
  • 参数：
    • 线程ID（pthread_t）。
    • 线程属性（可以为 NULL，表示默认属性）。
    • 线程函数（线程启动时执行的函数）。
    • 线程参数（传递给线程函数的参数）。
  • 返回值：成功时返回 0，失败时返回错误码。

2. pthread_exit()

  • 功能：结束调用线程的执行，并可以返回一个值给其他线程。
  • 参数：返回值（可以为 NULL）。
  • 注意：如果主线程调用此函数，整个进程会终止。

3. pthread_join()

  • 功能：等待指定线程结束，获取其返回值。
  • 参数：
    • 线程ID（pthread_t）。
    • 指向返回值的指针（可以为 NULL）。
  • 返回值：成功时返回 0，失败时返回错误码。

4. pthread_detach()

  • 功能：将线程设置为“分离状态”，使其在结束时自动释放资源。
  • 参数：线程ID（pthread_t）。
  • 注意：一旦线程被分离，无法再调用 pthread_join()。

5. pthread_cancel()

  • 功能：请求取消指定的线程。
  • 参数：线程ID（pthread_t）。
  • 注意：线程必须在可取消状态下，才能被取消。

6. pthread_mutex_lock() 和 pthread_mutex_unlock()

  • 功能：用于加锁和解锁互斥量，确保对共享资源的安全访问。
  • 参数：互斥量（pthread_mutex_t）。
  • 返回值：成功时返回 0，失败时返回错误码。

7. pthread_cond_wait() 和 pthread_cond_signal()

  • 功能：
    • pthread_cond_wait()：使线程等待条件变量，并自动释放互斥量。
    • pthread_cond_signal()：通知等待该条件变量的线程。
  • 参数：
    • pthread_cond_wait()：条件变量和互斥量。
    • pthread_cond_signal()：条件变量。

8. pthread_self()

  • 功能：获取当前线程的线程ID。
  • 返回值：返回当前线程的 pthread_t。

9. pthread_getschedparam() 和 pthread_setschedparam()

  • 功能：获取或设置线程的调度策略和优先级。
  • 参数：
    • 线程ID（pthread_t）。
    • 指向调度参数的结构体。

总结

这些线程相关的系统调用提供了创建、管理和同步线程的基本功能，是多线程编程的核心。正确使用这些调用能帮助开发者有效地利用多核处理器，提高程序的性能和响应能力。

IPC之信号
理论 2024/9/20

1. 信号的定义

  • 信号是一种异步通知机制，用于向进程发送事件通知。这些事件可以是外部事件（如用户按键、定时器到期）或内核事件（如进程错误）。

Each signal has a current disposition, which determines how the process behaves when it is delivered the signal.

2. 信号的类型

  • 常见的信号包括：
    • SIGINT：中断信号，通常由 Ctrl+C 发送。
    • SIGTERM：终止信号，要求进程终止。
    • SIGKILL：强制终止信号，不能被捕获或忽略。
    • SIGUSR1 和 SIGUSR2：用户定义的信号，可以用于自定义事件处理。

3. 信号的发送

  • 信号可以由以下方式发送：
    • 命令行：使用 kill 命令发送信号。kill -l 查看所有信号
    • 系统调用：使用 kill() 或 raise() 函数在程序内部发送信号。

进程相关系统调用
理论 2024/9/19

系统调用

1. fork()

  • 作用：创建一个新进程，复制当前进程的所有属性。
  • 返回值：在父进程中返回新创建的子进程的 PID，在子进程中返回 0，失败时返回 -1。
  • 在fork之前打开的文件对象是共享的，pc指针和代码段也是相同的

2. exec()

  • 作用：用新程序替换当前进程的映像。
  • 常用变体：execl(), execp(), execv() 等。
  • 返回值：成功时不返回，失败时返回 -1。

3. wait() / waitpid()

  • 作用：让父进程等待子进程结束，并获取子进程的退出状态。
  • 返回值：成功时返回结束的子进程的 PID，失败时返回 -1。
  • 参数NULL表示父进程不关心子进程的退出状态信息

4. exit()

  • 作用：终止当前进程，并返回状态给父进程。
  • 返回值：不返回。
  • 其它退出方式：_exit() : 不清理缓冲， return清理缓冲区，被信号中止， 主动调用 abort()

进程相关命令
理论 2024/9/18

命令

ps

  • 功能：显示当前运行的进程。

  • 常用选项

    • ps aux：显示所有用户的进程。
      • 显示所有用户的进程。
      • 输出格式较简洁，包含用户、PID、CPU 和内存使用率、运行时间、TTY、状态和命令等。
    • ps -elf：显示详细的进程信息。

      • 显示更详细的进程信息。

      • 输出包含更多列，如进程标志位、优先级、nice 值、内存地址、等待事件等

      • 输出列解释

        1. F：进程的标志位，表示该进程的状态和调度信息。（1-running，4-uninterruptable）

        2. S：进程的状态（例如，R 表示运行，S 表示休眠，Z 表示僵尸等。T暂停）。

        3. UID：进程所有者的用户 ID。

        4. PID：进程 ID，唯一标识进程。

        5. PPID：父进程 ID，表示创建此进程的进程。

        6. C：进程占用CPU的百分比

        7. PRI：进程的实际调度优先级，值越小，优先级越高，OS会根据该值来决定哪个进程可以获得cpu时间，该值会根据进程行为（如cpu使用情况，等待时间）自动调整

        8. NI：nice 值，范围通常在-20~19，数值越小优先级越高，较高的NI值会使PRI值上升从而降低其调度优先级

        9. ADDR：进程在内存中的地址（通常为 ?，表示不适用）。

        10. SZ：进程使用的内存大小，单位为页。

        11. VSZ：该进程使用的虚拟内存量，单位KB

        12. RSS：该进程占用的固定内存量，驻留中页的数量，单位KB

        13. WCHAN：进程正在等待的事件。

        14. STAT： D //无法中断的休眠状态（通常 IO 的进程）； R //正在运行可中在队列中可过行的； S //处于休眠状态； T //停止或被追踪； W //进入内存交换 （从内核2.6开始无效）； X //死掉的进程 （基本很少见）； Z //僵尸进程； < //优先级高的进程 N //优先级较低的进程 L //有些页被锁进内存； s //进程的领导者（在它之下有子进程）； l //多线程，克隆线程（使用 CLONE_THREAD, 类似 NPTL pthreads）； + //位于后台的进程组；

          linux是一个多用户，多任务的系统，可以同时运行多个用户的多个程序，就必然会产生很多的进程，而每个进程会有不同的状态。

        15. TTY：该进程在哪个终端上运行，若与终端无关则显示？。若显示pts/0则表示由网络连接主机进程

        16. TIME：进程使用的 CPU 时间。

        17. CMD：启动进程的命令行。

  • 补充—PRI（优先级）和 NI（nice 值）主要在Completely Fair Schedular下工作：

    完全公平调度器（CFS）

    • 描述：CFS 是 Linux 2.6.23 版本引入的默认调度器，旨在实现公平的 CPU 时间分配。
    • 工作原理
      • CFS 使用虚拟运行时间来衡量每个进程的运行时间。
      • NI 值会影响进程的权重，从而影响其在调度中的优先级。
      • PRI 值在 CFS 中并不直接使用，而是通过 NI 值间接影响调度。
