MySQL的一行记录是如何存储的
理论 2024/10/15
MySQL的数据存放在哪个文件？

大家都知道mysql的数据时存在磁盘的，那具体时保存在哪个文件呢？它的存储行为是由存储引擎实现的，mysql支持多种引擎，不同的引擎保存的文件自然也不同，本次主要讨论InnoDB的存储方式

先来看mysql数据库的文件存放在哪个目录？

mysql> show variables like 'datadir';
+---------------+-----------------+
| Variable_name | Value           |
+---------------+-----------------+
| datadir       | /var/lib/mysql/ |
+---------------+-----------------+
1 row in set (0.35 sec)

我们每创建一个database，都会在这个目录里面创建同名的目录，然后保存表结构和表数据的文件都会存放在这里，每个数据库下的各表都有对应的独占表空间文件---- xxx.ibd 文件，表数据就存在这里（5.6.6开始）

表空间文件的结构是怎么样的？

表空间是由 段segment，区extent，页page，行row组成，InnoDB存储引擎的逻辑存储结构如图

1.行 row

数据库表中的记录都是按行进行存放的，每行记录根据不同的行格式，有不同的存储结构

2. 页 page

记录是按照行来存储的，但是数据库的读取并不以【行】为单位，否则一次读取（也就是一次I/O）只能处理一行数据，效率会非常低。因此innodb的数据是按照页为单位读写的， 即需要读一条记录时，并不是将这个行记录从磁盘读出来，而是以页为单位，将其整体读入内存。默认每个页的大小为16kb，也就是最多能保证16kb的连续存储空间，页是innodb存储引擎磁盘管理的最小单元，这意味着一次IO最少操作16kb的内容，页的类型有很多，常见的有数据页，undo日志页，溢出页等，数据表中的行记录是用【数据页】来管理的，( •̀ ω •́ )也就是说，表中的记录存储在【数据页】里面

3. 区 extent

我们知道innodb是用B+树来组织数据的。

B+树中每一层都是通过双向链表连接起来的，如果以页为单位来分配存储空间，那么链表中相邻的两个页之间的物理位置并不是连续的，可能离得非常远，那么磁盘查询时就会有大量的随机IO，随机IO是非常慢的。

如何解决这个问题呢？只需让链表中相邻的页的物理位置也相邻，这样就可以使用顺序IO了，那么在范围查询（扫描叶子结点）的时候性能就会很高

那具体怎么解决呢？

在表中数据量大的时候，为某个索引分配空间的时候不再按照页为单位分配了。而是按照区（extent）为单位分配，每个区的大小为1MB，对于16kb的页来说，连续的64个页会被划为一个区，这样就使得链表中相邻的页的物理位置也相邻

4. 段 segment

表空间是由各个段 segment组成的，段是由多个区extent组成的，段一般分为数据段，索引段和回滚段等。

  • 索引段：存放B+树的非叶子节点的区的集合
  • 数据段：存放B+树的叶子节点的区的集合
  • 回滚段：存放的时回滚数据的区的集合，MVCC就是利用了回滚段实现了多版本查询数据。
INNODB行格式有哪些？

Innodb提供了四种行格式 row_format：分别是 Redundant, Compact, Dynamic, Compressed 行格式

这里我们只介绍一种 ，COMPACT格式，它和Dynamic格式非常像，后者是Mysql默认的行格式，弄懂了compact，其他格式也能很快看懂

可以看到，一条完整的记录分为 【记录的额外信息】和【记录的真实数据】两部分

epoll-tutor
理论 2024/9/30

使用 epoll 来完成 IO 多路复用

1. 创建 epoll 实例

  • 使用 epoll_create1() 创建一个 epoll 实例。内部有一个监听集合（红黑树）和就绪集合（线性表）

  • 数据结构：

    • epoll_fd：返回的文件描述符，代表 epoll 实例。

2. 注册文件描述符

  • 操作：

    • 使用

      int epoll_ctl(int epfd, int op, int fd, struct epoll_event *event);

      函数注册文件描述符，指定要监视的事件。

      • epfd：由 epoll_create1 返回的文件描述符。找到对应的监听集合（红黑树）
      • op：操作类型，可以是 EPOLL_CTL_ADD（添加）、EPOLL_CTL_MOD（修改）或 EPOLL_CTL_DEL（删除）。你希望对红黑树进行的操作
      • fd：指明了要进行操作的文件描述符👈
      • event：指向 struct epoll_event 的指针，指定或返回事件信息。前面op参数如果是删除操作，此项为NULL

  • 数据结构：

    • struct epoll_event：包含要监视的事件类型和用户数据（如文件描述符）。

      struct epoll_event {
    uint32_t events; // 事件类型，描述fd何时就绪
	//EPOLLET 边缘触发就绪，默认水平触发
    epoll_data_t data; // 就绪后，往就绪集合放入的数据
};
//用户数据定义如下
typedef union {
        void *ptr;      // 用户自定义指针
        int fd;         // 文件描述符
        uint32_t u32;   // 32位无符号整数
        uint64_t u64;   // 64位无符号整数
} epoll_data_t;

    • events：例如 EPOLLIN(可读时就绪)、EPOLLOUT(可写时就绪) 等。

    • data：可以存储与文件描述符相关的用户数据（如指针或整型值）。

3. 等待事件

  • 操作：

    • 使用

      int epoll_wait(int epfd, struct epoll_event *events, int maxevents, int timeout);

      函数等待事件发生。

      • epfd：epoll 实例的文件描述符。
      • events：用于存储发生的事件的数组。
      • maxevents：数组的大小，即可以存储的最大事件数。
      • timeout：超时时间（毫秒），如果大于0，表示等待的毫秒数，小于0表示一直等到有事件发生，等于0表示立即返回
      • 返回值：返回就绪集合实际的长度。

  • 数据结构：

    • struct epoll_event 结果数组：用于存储返回的就绪集合。
      • 每次调用 epoll_wait()，填充该数组（本质是拷贝内核态的数据一份），返回就绪的事件数量。

TCP
理论 2024/9/25

TCP基本认识

1. 看一看TCP的头

Sequence_Number: 在建立连接时由计算机生成的随机数作为初始值，通过SYN包传给接收端主机，每发送一次数据，累加 一次该 数据字节数 的大小。用来解决网络包乱序问题。

Ackonwledge_Number: 指下一次 期望 收到的数据的序列号，发送端收到这个确认应答以后可以认为这个序号以前的数据都已经被正常接收。用来解决丢包的问题。

flag：

  • ACK：该位为1时，确认应答 的字段变为有效，TCP规定处理最初建立连接时的SYN包之外该位必须设置为1
  • RST： 该位为1时，表示TCP连接中出现异常必须强制断开连接
  • SYN：该位为1时，表示希望建立连接，并在其 序列号 的字段进行序列号初始值的设定
  • FIN：该位为1时，表示今后不会再有数据发送，希望断开连接。当通信结束希望断开连接时，通信双方的主机之间就可以相互交换FIN位为1的TCP段。

线程
理论 2024/9/20

基本概念

linux中线程又叫做轻量级进程（light-weight process LWP），也有PCB，创建线程使用的底层函数和进程底层一样，都是clone，但没有独立的地址空间；而进程有独立地址空间，拥有PCB。
Linux下：线程是最小的执行单位，调度的基本单位。进程是最小分配资源单位，可看成是只有一个线程的进程。
线程是一个进程内部的控制序列。控制序列可以理解为一个执行流。进程内部是指虚拟地址空间。

1. pthread_create()

  • 功能：创建一个新线程。
  • 参数：
    • 线程ID（&pthread_t）。传入线程id的指针
    • 线程属性（可以为 NULL，表示默认属性）。
    • 线程函数（线程启动时执行的函数）。
    • 线程参数（传递给线程函数的参数）。
  • 返回值：成功时返回 0，失败时返回错误码。
  • 注意：第三方库, 需要手动链接 -lpthread
