  2. 分类

    • 排序

  3. for_each 算法

    // Apply a function to every element of a sequence. 
for_each(v.begin(), v.end(), [a](int v)mutable
    {
        --a;
        cout << "a=" << a << endl;
        v=v*2;
        cout << v << " ";
    });
//对容器内的所有元素执行此匿名函数
//[] 为捕获列表，默认空表示不捕获， = 表示捕获所有变量（按值）
//& 表示捕获所有变量（按引用）。 mutable 表 你可以修改按值捕获的变量（read-only）的副本

  4. remove_if 算法

    std::vector<int> v{1, 2, 3, 4, 5, 6};

// 移除所有大于 3 的元素
auto it = std::remove_if(v.begin(), v.end(), [](int x) { return x > 3; });
v.erase(it, v.end());
// 注意
//1.不是真正删除：
//std::remove_if 仅重新排列容器中的元素，逻辑上删除的元素仍然存在于容器中，必须使用 erase 方法来真正删除它们。
//2.返回迭代器：
//remove_if 返回一个指向新逻辑结尾的迭代器，后续的 erase 调用应使用该迭代器。
//3.适用容器：
//remove_if 可以与所有支持随机访问迭代器的容器（如 vector、deque 和 array）结合使用，但不适用于 list，因为 remove_if 只能使用随机访问迭代器。

标准模板库概述

STL的六大组件

  1. 容器：用来存放数据的，数据结构

    • 序列式容器 vector, list, deque
    • 关联式容器 set, map
    • 无序关联式容器 unordered_set, unordered_map

  2. 迭代器：看成是一种指针，广义指针（具备指针的功能）。可以访问容器中的元素

  3. 算法：操作容器中的元素

  4. 适配器：起到适配的作用

    • 容器适配器 stack，queue, priority_queue

      (容器适配器是对其他容器（如 std::deque、std::list）的封装，它们提供了不同的接口，以实现特定的数据结构行为。适配器不直接存储数据，而是通过组合已有的容器来实现某种数据结构的特性。)

    • 迭代器适配器 front_inserter, back_inserter

      (迭代器适配器可以改变基本迭代器的行为，使其能够适应不同的需求。例如，front_inserter 将元素插入到容器的前端，而普通的迭代器可能只是用于访问或修改元素)

    • 函数适配器

  5. 函数对象（仿函数）：定制化操作

  6. 空间配置器：进行空间的申请与释放操作 （使用+原理+源码）

概念区分

重载 overload：发生在同一个类中，当函数名称相同时，函数参数类型，顺序，个数任一不同

隐藏 oversee：发生在基类和派生类之间，函数名称相同时，就构成隐藏（参数不同也能构成隐藏）

覆盖 override：发生在基类和派生类之间，基类和派生类中同时定义相同的虚函数，覆盖的是虚函数表中的入口地址，并不知覆盖函数本身

C++中有哪几种多态？请详细说明一下

  1. 静态多态：包括函数重载，运算符重载，模板，在编译时发生
  2. 动态多态：使用虚函数实现，在运行时发生，当类中定义了虚函数之后，在对象的存储开始位置，多一个虚函数指针，指向一张虚函数表，表中存储的是虚函数入口地址，如果此类有派生类，且派生类中对虚函数进行了覆盖，对于派生类而言，将会覆盖表中的虚函数入口地址

智能指针的实现原理是什么？用到了什么技术？该技术有哪些特征？

智能指针是 C++ 中一种用于管理动态内存的对象，智能指针利用 RAII 技术，确保资源在对象的生命周期内得到管理。对象的构造和析构与资源的获取和释放紧密相连。还使用了模板，以支持不同类型的资源。

C++ 标准库提供了几种智能指针的实现，最常用的有：

  • std::unique_ptr：独占所有权，不能被复制，但可以被移动。
  • std::shared_ptr：共享所有权，多个指针可以指向同一个对象，通过引用计数来管理对象的生命周期。当引用计数为零时，资源被释放。
  • std::weak_ptr：与 shared_ptr 结合使用，持有对对象的弱引用，不增加引用计数，避免循环引用。

移动语义

  • std::unique_ptr 支持移动语义，意味着它的所有权可以被转移，但不能被复制。
  • 移动构造函数和移动赋值运算符被实现为简单的指针转移，并将原对象的指针置为 nullptr，确保不会双重释放内存。

什么是右值引用？C++11为什么要引入右值引用？

在c++中，右值是指哪些没有持久地址的临时对象，通常出现在表达式的右侧。右值引用则是一个新的引用类型，用于绑定到右值，其语法为 && ，与左值引用的& 相对。

因为要优化资源管理和提高性能，在之前我们把一个临时的类对象赋值给变量时，通常会调用类的构造与赋值函数，这样做就涉及到对象的创建和销毁，浪费性能，所以引入右值和移动语义，移动语义指的是通过“移动”而非“复制”的方式来管理资源。这意味着可以将资源的所有权从一个对象转移到另一个对象，而不需要复制资源本身。其实现依赖于右值引用，允许开发者定义移动构造函数和移动赋值运算符，从而在对象间安全地转移资源。

int&& rvalueRef = 42; // 42 是一个右值，rvalueRef 是一个右值引用

MySQL的一行记录是如何存储的
理论 2024/10/15
MySQL的数据存放在哪个文件？

大家都知道mysql的数据时存在磁盘的，那具体时保存在哪个文件呢？它的存储行为是由存储引擎实现的，mysql支持多种引擎，不同的引擎保存的文件自然也不同，本次主要讨论InnoDB的存储方式

先来看mysql数据库的文件存放在哪个目录？

mysql> show variables like 'datadir';
+---------------+-----------------+
| Variable_name | Value           |
+---------------+-----------------+
| datadir       | /var/lib/mysql/ |
+---------------+-----------------+
1 row in set (0.35 sec)

我们每创建一个database，都会在这个目录里面创建同名的目录，然后保存表结构和表数据的文件都会存放在这里，每个数据库下的各表都有对应的独占表空间文件---- xxx.ibd 文件，表数据就存在这里（5.6.6开始）

表空间文件的结构是怎么样的？

表空间是由 段segment，区extent，页page，行row组成，InnoDB存储引擎的逻辑存储结构如图

1.行 row

数据库表中的记录都是按行进行存放的，每行记录根据不同的行格式，有不同的存储结构

2. 页 page

记录是按照行来存储的，但是数据库的读取并不以【行】为单位，否则一次读取（也就是一次I/O）只能处理一行数据，效率会非常低。因此innodb的数据是按照页为单位读写的， 即需要读一条记录时，并不是将这个行记录从磁盘读出来，而是以页为单位，将其整体读入内存。默认每个页的大小为16kb，也就是最多能保证16kb的连续存储空间，页是innodb存储引擎磁盘管理的最小单元，这意味着一次IO最少操作16kb的内容，页的类型有很多，常见的有数据页，undo日志页，溢出页等，数据表中的行记录是用【数据页】来管理的，( •̀ ω •́ )也就是说，表中的记录存储在【数据页】里面

3. 区 extent

我们知道innodb是用B+树来组织数据的。

B+树中每一层都是通过双向链表连接起来的，如果以页为单位来分配存储空间，那么链表中相邻的两个页之间的物理位置并不是连续的，可能离得非常远，那么磁盘查询时就会有大量的随机IO，随机IO是非常慢的。

如何解决这个问题呢？只需让链表中相邻的页的物理位置也相邻，这样就可以使用顺序IO了，那么在范围查询（扫描叶子结点）的时候性能就会很高

那具体怎么解决呢？

在表中数据量大的时候，为某个索引分配空间的时候不再按照页为单位分配了。而是按照区（extent）为单位分配，每个区的大小为1MB，对于16kb的页来说，连续的64个页会被划为一个区，这样就使得链表中相邻的页的物理位置也相邻

4. 段 segment

表空间是由各个段 segment组成的，段是由多个区extent组成的，段一般分为数据段，索引段和回滚段等。

  • 索引段：存放B+树的非叶子节点的区的集合
  • 数据段：存放B+树的叶子节点的区的集合
  • 回滚段：存放的时回滚数据的区的集合，MVCC就是利用了回滚段实现了多版本查询数据。
INNODB行格式有哪些？

Innodb提供了四种行格式 row_format：分别是 Redundant, Compact, Dynamic, Compressed 行格式

这里我们只介绍一种 ，COMPACT格式，它和Dynamic格式非常像，后者是Mysql默认的行格式，弄懂了compact，其他格式也能很快看懂

可以看到，一条完整的记录分为 【记录的额外信息】和【记录的真实数据】两部分

epoll-tutor
理论 2024/9/30

使用 epoll 来完成 IO 多路复用

1. 创建 epoll 实例

  • 使用 epoll_create1() 创建一个 epoll 实例。内部有一个监听集合（红黑树）和就绪集合（线性表）

  • 数据结构：

    • epoll_fd：返回的文件描述符，代表 epoll 实例。

2. 注册文件描述符

  • 操作：

    • 使用

      int epoll_ctl(int epfd, int op, int fd, struct epoll_event *event);

      函数注册文件描述符，指定要监视的事件。

      • epfd：由 epoll_create1 返回的文件描述符。找到对应的监听集合（红黑树）
      • op：操作类型，可以是 EPOLL_CTL_ADD（添加）、EPOLL_CTL_MOD（修改）或 EPOLL_CTL_DEL（删除）。你希望对红黑树进行的操作
      • fd：指明了要进行操作的文件描述符👈
      • event：指向 struct epoll_event 的指针，指定或返回事件信息。前面op参数如果是删除操作，此项为NULL

  • 数据结构：

    • struct epoll_event：包含要监视的事件类型和用户数据（如文件描述符）。

      struct epoll_event {
    uint32_t events; // 事件类型，描述fd何时就绪
	//EPOLLET 边缘触发就绪，默认水平触发
    epoll_data_t data; // 就绪后，往就绪集合放入的数据
};
//用户数据定义如下
typedef union {
        void *ptr;      // 用户自定义指针
        int fd;         // 文件描述符
        uint32_t u32;   // 32位无符号整数
        uint64_t u64;   // 64位无符号整数
} epoll_data_t;

    • events：例如 EPOLLIN(可读时就绪)、EPOLLOUT(可写时就绪) 等。

    • data：可以存储与文件描述符相关的用户数据（如指针或整型值）。

3. 等待事件

  • 操作：

    • 使用

      int epoll_wait(int epfd, struct epoll_event *events, int maxevents, int timeout);

      函数等待事件发生。

      • epfd：epoll 实例的文件描述符。
      • events：用于存储发生的事件的数组。
      • maxevents：数组的大小，即可以存储的最大事件数。
      • timeout：超时时间（毫秒），如果大于0，表示等待的毫秒数，小于0表示一直等到有事件发生，等于0表示立即返回
      • 返回值：返回就绪集合实际的长度。

  • 数据结构：

    • struct epoll_event 结果数组：用于存储返回的就绪集合。
      • 每次调用 epoll_wait()，填充该数组（本质是拷贝内核态的数据一份），返回就绪的事件数量。
