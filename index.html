epoll-tutor
理论 2024/9/30

使用 epoll 来完成 IO 多路复用

1. 创建 epoll 实例

  • 操作：
    • 使用 epoll_create1() 创建一个 epoll 实例。
  • 数据结构：
    • epoll_fd：返回的文件描述符，代表 epoll 实例。

2. 注册文件描述符

  • 操作：

    • 使用

      int epoll_ctl(int epfd, int op, int fd, struct epoll_event *event);

      函数注册文件描述符，指定要监视的事件。

      • epfd：由 epoll_create1 返回的文件描述符。
      • op：操作类型，可以是 EPOLL_CTL_ADD（添加）、EPOLL_CTL_MOD（修改）或 EPOLL_CTL_DEL（删除）。
      • fd：指明了要进行操作的文件描述符👈
      • event：指向 struct epoll_event 的指针，指定或返回事件信息。里面的data字段存储与上面☝🏻文件描述符相关的数据

  • 数据结构：

    • struct epoll_event：包含要监视的事件类型和用户数据（如文件描述符）。

      struct epoll_event {
    uint32_t events; // 事件类型
    epoll_data_t data; // 用户数据
};
//用户数据定义如下
typedef union {
        void *ptr;      // 用户自定义指针
        int fd;         // 文件描述符
        uint32_t u32;   // 32位无符号整数
        uint64_t u64;   // 64位无符号整数
} epoll_data_t;

    • events：例如 EPOLLIN(可读时就绪)、EPOLLOUT(可写时就绪) 等。

    • data：可以存储与文件描述符相关的用户数据（如指针或整型值）。

3. 等待事件

  • 操作：

    • 使用

      int epoll_wait(int epfd, struct epoll_event *events, int maxevents, int timeout);

      函数等待事件发生。

      • epfd：epoll 实例的文件描述符。
      • events：用于存储发生的事件的数组。
      • maxevents：数组的大小，即可以存储的最大事件数。
      • timeout：超时时间（毫秒），如果大于0，表示等待的毫秒数，小于0表示一直等到有事件发生，等于0表示立即返回
      • 返回值：返回就绪事件的数量。

  • 数据结构：

    • struct epoll_event 结果数组：用于存储返回的事件信息。
      • 每次调用 epoll_wait()，填充该数组，返回发生的事件数量。本质是拷贝内核态的数据一份

TCP
理论 2024/9/25

TCP基本认识

1. 看一看TCP的头

Sequence_Number: 在建立连接时由计算机生成的随机数作为初始值，通过SYN包传给接收端主机，每发送一次数据，累加 一次该 数据字节数 的大小。用来解决网络包乱序问题。

Ackonwledge_Number: 指下一次 期望 收到的数据的序列号，发送端收到这个确认应答以后可以认为这个序号以前的数据都已经被正常接收。用来解决丢包的问题。

flag：

  • ACK：该位为1时，确认应答 的字段变为有效，TCP规定处理最初建立连接时的SYN包之外该位必须设置为1
  • RST： 该位为1时，表示TCP连接中出现异常必须强制断开连接
  • SYN：该位为1时，表示希望建立连接，并在其 序列号 的字段进行序列号初始值的设定
  • FIN：该位为1时，表示今后不会再有数据发送，希望断开连接。当通信结束希望断开连接时，通信双方的主机之间就可以相互交换FIN位为1的TCP段。

线程
理论 2024/9/20

基本概念

linux中线程又叫做轻量级进程（light-weight process LWP），也有PCB，创建线程使用的底层函数和进程底层一样，都是clone，但没有独立的地址空间；而进程有独立地址空间，拥有PCB。 Linux下：线程是最小的执行单位，调度的基本单位。进程是最小分配资源单位，可看成是只有一个线程的进程。 线程是一个进程内部的控制序列。控制序列可以理解为一个执行流。进程内部是指虚拟地址空间。

1. pthread_create()

  • 功能：创建一个新线程。
  • 参数：
    • 线程ID（&pthread_t）。传入线程id的指针
    • 线程属性（可以为 NULL，表示默认属性）。
    • 线程函数（线程启动时执行的函数）。
    • 线程参数（传递给线程函数的参数）。
  • 返回值：成功时返回 0，失败时返回错误码。
  • 注意：第三方库, 需要手动链接 -lpthread

IPC之信号
理论 2024/9/20

1. 信号的定义

  • 信号是一种异步通知机制，用于向进程发送事件通知。这些事件可以是外部事件（如用户按键、定时器到期）或内核事件（如进程错误）。

Each signal has a current disposition, which determines how the process behaves when it is delivered the signal.

2. 信号的类型

  • 常见的信号包括：
    • SIGINT：中断信号，通常由 Ctrl+C 发送。
    • SIGTERM：终止信号，要求进程终止。
    • SIGKILL：强制终止信号，不能被捕获或忽略。
    • SIGUSR1 和 SIGUSR2：用户定义的信号，可以用于自定义事件处理。

3. 信号的发送

  • 信号可以由以下方式发送：
    • 命令行：使用 kill 命令发送信号。kill -l 查看所有信号
    • 系统调用：使用 kill() , abort() raise() 函数在程序内部发送信号。

进程相关系统调用
理论 2024/9/19

系统调用

1. fork()

  • 作用：创建一个新进程，复制当前进程的所有属性。
  • 返回值：在父进程中返回新创建的子进程的 PID，在子进程中返回 0，失败时返回 -1。
  • 在fork之前打开的文件对象是共享的，pc指针和代码段也是相同的

2. exec()

  • 作用：用新程序替换当前进程的映像。
  • 常用变体：execl(), execp(), execv() 等。
  • 返回值：成功时不返回，失败时返回 -1。

3. wait() / waitpid()

  • 作用：让父进程等待子进程结束，并获取子进程的退出状态。
  • 返回值：成功时返回结束的子进程的 PID，失败时返回 -1。
  • 参数NULL表示父进程不关心子进程的退出状态信息

4. exit()

  • 作用：终止当前进程，并返回状态给父进程。
  • 返回值：不返回。
  • 其它退出方式：_exit() : 不清理缓冲， return清理缓冲区，被信号中止， 主动调用 abort()
